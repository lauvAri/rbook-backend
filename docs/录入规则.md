# R 代码录入规范指南  

> **核心理念**：您只需专注**统计分析逻辑**和**绘图内容**，所有技术细节（文件路径、设备开关、包安装）均由后端自动处理。

---

## 一、三大核心机制（必读）

###  1. 文件输入：使用 `input_file` 变量
| 您要做的                               | 后端自动处理                                                              |
| ---------------------------------- | ------------------------------------------------------------------- |
| ```data <- read.csv(input_file)``` | • 自动创建 `data.csv`<br>• 自动注入 `input_file <- "data.csv"`<br>• 无需硬编码路径 |
| **禁止写法**                           |  **正确写法**                                                           |
| `read.csv("mydata.csv")`           | `read.csv(input_file)`                                              |
| `setwd("/home/user")`              | （完全不需要）                                                             |

---

### 2. 变量替换：直接使用预定义变量

后端自动注入变量，**无需声明/赋值**：

**录入规则**：
```r
# 正确：直接使用变量（无需赋值）
result <- chisq.test(table(data$group, data$x), correct = yates_correction)

# 错误：手动赋值（会被后端覆盖）
yates_correction <- TRUE  # ← 删除此行！
```

>  **重要**：变量名由教学设计人员提前定义，录入时**严格按文档名称书写**（区分大小写）。

---

###  3. 绘图机制：每条绘图命令 = 1张独立图片
| 您要做的                               | 后端自动处理                                                                                                                     |
| ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `barplot(...)`<br>`plot(...)`        | • 自动执行 `png("plot_%03d.png")`<br>• 每个绘图命令生成独立图片（plot_001.png, plot_002.png...）<br>• 自动执行 `dev.off()`<br>• 自动转为 Base64 返回前端 |
| **禁止写法**                           |  **正确写法**                                                                                                                  |
| `png("myplot.png")`<br>`dev.off()` | （完全不需要）<br>直接写 `barplot(...)`                                                                                              |

**多图输出示例**：
```r
# 图1：自动保存为 plot_001.png
barplot(table(data$group, data$x), main = "频数分布")

# 图2：自动保存为 plot_002.png
boxplot(value ~ group, data = data, main = "箱线图")

# 图3：自动保存为 plot_003.png
plot(data$x, data$y, main = "散点图")
```

> **关键规则**：  
> - 每个独立的绘图命令（`barplot`/`plot`/`hist`等）自动生成**1张图片**  
> - **基础R绘图**无需 `print()`（函数本身会触发渲染）  
> - **ggplot2 绘图必须 `print()`**，否则不会输出到图片  
> - **禁止**调用 `dev.off()`（会导致后续图片丢失）

## 二、标准代码模板（复制即用）

```r
# ========== 1. 数据加载（固定写法）==========
data <- read.csv(input_file)

# ========== 2. 业务逻辑（按教学需求编写）==========
# 示例：卡方检验（使用预定义变量 yates_correction）
tbl <- table(data$group, data$x)
result <- chisq.test(tbl, correct = yates_correction)

# ========== 3. 结果输出（控制台文本）==========
cat("卡方值:", round(result$statistic, 3), "\n")
cat("p值:", format.pval(result$p.value, digits = 3), "\n")

# ========== 4. 可视化（每条命令 = 1张图）==========
# 图1：分组条形图
barplot(tbl, beside = TRUE,
        col = c("skyblue", "salmon"),
        main = "频数分布",
        xlab = "组别", ylab = "频数")

# 图2：堆叠比例图
barplot(prop.table(tbl, 1) * 100,
        col = c("skyblue", "salmon"),
        main = "组内比例",
        ylab = "百分比(%)")

# 图3：马赛克图（列联表结构）
mosaicplot(tbl, main = "列联表结构", color = TRUE)
```

## 三、高级功能：使用 ggplot2 等扩展包

### 正确做法（后端自动安装）
```r
# 1. 声明依赖（仅需 library，无需 install.packages）
library(ggplot2)

# 2. 正常绘图（自动生成第4张图）
p <- ggplot(data, aes(x = group, fill = x)) +
  geom_bar(position = "dodge") +
  labs(title = "ggplot2美化图")
print(p)
```

### 录入注意事项
| 事项          | 说明                                                       |
| ----------- | -------------------------------------------------------- |
| **包名提取**    | 后端通过正则自动检测 `library(包名)`，**必须写全称**（如 `library(ggplot2)`） |
| **无需安装代码**  | 删除所有 `install.packages()` 调用                             |
| **首次执行稍慢**  | 首次运行会自动安装包（约30-60秒），后续秒级响应                               |
| **失败不影响前图** | 即使 ggplot2 安装失败，前3张基础R图片仍会正常返回                           |
| **必须显式渲染**  | ggplot2 在非交互脚本中需要 `print(p)` 才会输出图片                    |

## 四、最佳实践总结

| 场景       | 推荐做法                         | 禁止做法                    |
| -------- | ---------------------------- | ----------------------- |
| **数据加载** | `read.csv(input_file)`       | 硬编码路径/修改文件名             |
| **参数使用** | 直接使用 `yates_correction`      | 手动赋值覆盖                  |
| **基础绘图** | 每个 `barplot()`/`plot()` 单独成行 | 合并到同一行                  |
| **高级绘图** | `library(ggplot2)` + 正常绘图    | 手动 `install.packages()` |
| **设备管理** | **完全不写** `png()`/`dev.off()` | 任何设备开关代码                |
| **错误处理** | 无需 tryCatch（后端已封装）           | 添加冗余错误捕获                |
